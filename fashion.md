## 1. Application Type

This is a **Fashion E-Commerce Mobile Application**. Its primary purpose is to allow users to browse, discover, and purchase clothing and fashion items directly from their mobile device.

---

## 2. Design

The app utilizes a **Dark Mode/Black-centric design** for a sleek, modern, and high-end aesthetic.

### Color Scheme

**Primary Colors**:
- **Pure Black** is the dominant background color, providing a striking contrast.

**Secondary Colors**:
- **Lower/Off-Black** is used for secondary elements like input fields, buttons, and card backgrounds to provide depth and visual separation.

**Typography & UI Elements**:
- Text and key UI elements (icons, button text) are rendered in **White** for maximum contrast and readability against the black background.
- A subtle, **light grey/off-white** is also used for less important text (e.g., descriptions, input placeholders).

### Aesthetic
The design is **minimalist and sophisticated**, focusing on large, high-quality product imagery, which stands out prominently against the dark theme. The overall look is professional and geared towards a modern fashion consumer.

## 3. Key Features

### Authentication
- Includes dedicated **Sign Up** (Create Account) and **Sign In** pages for secure user access
- Features fields for **Name**, **Email**, and **Password**

### Product Discovery
- A dedicated **Search bar** is prominently displayed on the main product listing screen
- Products are organized into specific **Categories**:
  - New Arrivals
  - Tops
  - Bottom
  - Outerwear
  - Dresses
  - Skirts
  - etc.

### Shopping & Checkout
- A comprehensive **Checkout screen** is provided, detailing:
  - **Order Summary** (Subtotal, Shipping, Total)
  - Allows users to **Change Shipping Address**
  - Features a dedicated **Payment Method** section

### Payment Options
- Supports common payment methods:
  - **Credit Card (Visa)** input:
    - Card number
    - Expiry date
    - CVV
  - Secondary option for **PayPal** (Pay with Pral is shown, likely a placeholder for PayPal or a similar service)

### Personal Profile
- A dedicated **Profile page** allows the user to:
  - View their **name** and **email**
  - Access **Order History**
  - View **Favourites**
  - Manage **Account Settings**

### Order Tracking
- The Profile page includes a section for **Recent Orders** with:
  - Images
  - Status (e.g., 'Delivered Today')
  - Item names

### Wishlist/Favourites
- A **Favourites** section/tab in the profile allows users to easily view items they have previously selected but not yet purchased

### Navigation
- A consistent **Bottom Navigation Bar** is present across multiple screens
- Includes icons for:
  - **Home**
  - **Favourites**
  - **Cart**
  - **Profile**
- Enables easy switching between core app functions

## 4. User Journey

The user journey is intuitive and covers the full e-commerce flow:

### Step 1: Welcome & Access
- The user starts on a **Welcome/Splash Screen**
- Then proceeds to the **Create Account** (Sign Up) screen to register
- Or the **Log In** screen to sign in

### Step 2: Product Discovery
Once signed in, the user lands on the **Home** (or main product) screen, featuring a grid view of products.

- They can use the **Search Bar** or the **Category Tags** (New Arrivals, Tops, etc.) to browse

### Step 3: Selection & Cart
- The user taps on an item to view **details** (not shown, but implied)
- They add the desired item(s) to their **Cart**
- They can also mark items as **Favourites** for later purchase (accessed via the Favourites icon/tab)

### Step 4: Initiating Purchase
- The user proceeds to the **Cart**
- Then to the **Checkout** screen

### Step 5: Checkout Process
1. The user reviews the **Order Summary** (subtotals, shipping)
2. They confirm or **Change the Shipping Address**
3. They select or enter their **Payment Method** (e.g., inputting Credit Card details)
4. They click **"Complete Order"**

### Step 6: Post-Purchase & Account Management
After the order is placed, the user can navigate to their **Profile page**:

- Here, they can check their **order status** in Recent Orders or Order History
- They can revisit items saved in **Favourites**
- Finally, they can **Log Out** when finished
